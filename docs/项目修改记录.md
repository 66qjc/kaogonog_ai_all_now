## 第 1 轮：后端结构梳理与幻觉抑制增强

### 修改时间

2026-04-04

### 这一轮改动的目标

1. 修复后端中影响稳定性和可扩展性的核心问题。
2. 提升大模型评分结果的结构稳定性。
3. 降低“模型编造原话”“总分不一致”“维度乱写”等问题。
4. 把代码补成更适合弱基础学习者阅读的版本。

### 发现的主要问题

#### 1. 题目 ID 没有真正参与评分

虽然接口里传了 `question_id`，但原有流程实际上没有基于这个 ID 选择题目，导致接口参数与内部逻辑脱节。

#### 2. 视觉描述和考生原文被混在一起

原先视频处理后的文本把“视觉分析结论”和“语音转录内容”拼到一个字符串中直接送给大模型，容易导致模型把视觉观察误认为考生真的说过的话。

#### 3. 大模型输出缺少强校验

主要风险包括：

1. 返回未定义维度。
2. 分数超出满分。
3. 总分和分项得分不一致。
4. 引用了 transcript 中不存在的原话。

#### 4. 配置与路径解析不够稳定

项目在不同启动目录下运行时，题库路径可能出问题。

#### 5. 工程可维护性不够

1. 缺少完整的数据模型。
2. 依赖注入不集中。
3. 阅读门槛较高。

## 第 3 轮：河南省直定制评分校准

### 修改时间

2026-04-07

### 修改目标

1. 修复 `26分.txt` 这类“通用高分样本”被河南省直定制规则直接打入低分区的问题。
2. 保留“河南省情 + 省直岗位化”扣分能力，但避免重复放大。
3. 让接口层与真实评分链路回归验证通过。

### 修改内容

1. 调整第二阶段评分 Prompt，明确“先评通用内容质量，再评河南省直适配度”。
2. 在 Prompt 中加入分档锚点：
   - `24-30` 为河南省直高分
   - `20-23` 为通用高分但本土化不足
   - `16-19` 为基本合格
   - `0-15` 为明显偏弱
3. 收紧自动缺失证据规则：
   - 省情缺失和岗位视角缺失合并为一类缺失证据
   - 措施层次不足只在措施证据本身也不足时才补充
   - 口语化自动证据数量增加上限，避免重复刷负面证据
4. 在后处理增加“通用高分保底校准”：
   - 当答案已覆盖现象、危害、措施三大核心内容
   - 但仅缺河南省情/省直岗位化表达时
   - 若模型把它直接打入低分区，则回调到中上档
5. 新增针对该逻辑的单元测试。

## 第 12 轮：回归误差前三项定向优化

### 修改时间

2026-04-27

### 修改目标

1. 根据 `4.27` 全量回归分析与阶段 0 误判样例，优先处理几类高频问题：
   - 人际题中档程序化样本过于空泛，容易被打到不及格附近
   - 人际题阶段 2 评分对“基本合格”缺少稳定锚点
   - 维度别名不统一，导致 `language`、`通用` 等提示被当作未知维度清空
   - 阶段 0 违规检测过度依赖词面风险，容易把“废物利用”“不能走后门”“答题复盘语气”误判成违规
2. 继续采用“小批量真实大模型回归”验证，不重复跑整批 `545` 条样本。

### 修改内容

1. 在 [import_question_bank.py](../ai_gongwu_backend/scripts/import_question_bank.py) 重写人际题中档样本生成逻辑：
   - 扩充 `infer_target_group()` 的关系识别词
   - 新增 `_extract_dimension_names()`、`_build_interpersonal_focus_clauses()`、`_build_interpersonal_mid_openings()`、`_build_interpersonal_mid_closing()`
   - 让 `build_interpersonal_template_texts(..., mode="mid")` 从通用套话改为“对象关系 + 责任动作 + 解释澄清 + 跟进补救”的场景化生成
2. 在 [prompts.py](../ai_gongwu_backend/app/services/scoring/prompts.py) 给人际题阶段 2 增加“基本合格”判档护栏：
   - 新增 `_is_interpersonal_question()` 与 `_build_interpersonal_scoring_guardrail()`
   - 如果答案已做到“态度得体 + 解释原因/误会 + 后续动作”，即使不出彩，也不应直接打成 `0` 分
3. 在 [flow.py](../ai_gongwu_backend/app/services/flow.py) 与 [prompts.py](../ai_gongwu_backend/app/services/scoring/prompts.py) 把阶段 0 违规检测改造成“纯 LLM 上下文判定”：
   - 删除本地硬编码违规词短路，不再因为单个词面命中就直接 `0` 分
   - 阶段 0 改为先单独调用一次大模型专项审查，只在“明确辱骂 / 明确鼓吹走后门受贿 / 明确政治红线”等情形下终止评分
   - prompt 明确写入“废物利用”“不能走后门”“这类演讲题不用一开始铺得太满”等非违规反例，要求模型必须结合整句整段语义判断
   - `matched_terms` 从“关键词”收紧为“最能证明违规的原文短语或短句片段”
   - 阶段 0 调用失败时按 `fail-open` 放行继续评分，并在 `validation_notes` 中记录“阶段 0 违规检测失败，已放行继续评分。”
4. 在 [calculator.py](../ai_gongwu_backend/app/services/scoring/calculator.py) 增加维度别名归一：
   - 新增 `DIMENSION_ALIAS_HINTS`、`ENGLISH_DIMENSION_ALIAS_HINTS`
   - 在 `prepare_evidence_packet()` 与 `_normalize_reason_items()` 中补别名解析
   - 让 `language -> ⑤语言逻辑`、`通用 -> 根据理由上下文归一到目标维度` 这类情况优先保留而不是直接清空
5. 重新生成题库与回归样本：
   - `python scripts/import_question_bank.py --profile anhui`
   - `python scripts/import_question_bank.py --profile hunan`

### 验证情况

   - `python -X utf8 -m unittest tests.test_import_question_bank tests.test_scoring_calculator tests.test_scoring_flow`
   - 共 `23` 项测试通过
   - 其中覆盖了“明确辱骂仍需拦截”“废物利用/不能走后门/教学复盘语气不应误判”“阶段 0 失败按 fail-open 继续评分”等关键场景

   - 共 `12` 条样本，结果 `PASS 12 / FAIL 0 / ERROR 0`

   - `AHGWY-20210709-02` 的人际题中档样本由此前全量回归中的 `0.0` 失配，恢复到小批量回归中的 `11.0` 并落入期望区间
   - `AHGWY-20250517-02` 不再因“废物利用”这类上下文误判被阶段 0 直接打断，`mid / low` 样本均通过
   - 之前容易被“不能走后门”“教学讲解语气”误杀的样本，现已能进入正常证据抽取与评分链路
